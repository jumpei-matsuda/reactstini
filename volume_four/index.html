Firebaseの導入

パッケージのインストール

最終号ですが、サーバー接続を目標に進めていきます。今回はFirebaseを使用しますが、知識がある方であればSpringBootで実装したり、AWSを使用して頂いても大丈夫です。

Firebase自体の設定の前に、reactプロジェクトでfirebaseを使用できるようにパッケージをインストールしましょう。後ほど細かい設定を行いますので、今はインストールだけで大丈夫です。 

yarn add firebase
yarn add firebase-tools

Firebaseの設定

それではFirebaseの設定を行っていきます。まずFirebaseにアクセスしてみましょう。「プロジェクトを作成」を押下し任意のプロジェクト名を設定してください。

その後Googleアナリティクスの設定画面が表示されますが、今回は特に不要なのでチェックを外しても大丈夫です。そのまま右下の「プロジェクトを作成」ボタンを押下してしばらく待機すると作成が完了します。
「続行」を押下すると以下のような画面が表示されていることを確認してください。

続いて画面中央にある「</>（ウェブ）」というボタンからアプリの登録を行います。任意のニックネームを入力し、Firebase Hostingの設定にチェックをいれて「アプリを登録」をクリックします。

登録が完了すると、「Firebase SDK の追加」という項目にスクリプトが表示されるかと思いますが、一度左上の✕ボタンで画面を閉じ、続いてデータベースの設定を行います。左のツールバーの「構築」内にある「Firebase Database」を選択しましょう。今回は「Cloud Firestore」というドキュメント型のデータベースを使用します。

上記のような画面が表示されれば、真ん中辺りの「データベースの作成」を押下します。本番運用かテスト運用か選択するモーダルが表示されるかと思いますが、一度テスト運用を選択してデータベースを作成しましょう。しばらくするとデータベースが作成されコンソールに移動します。

続いて、Cloud Firestoreのロケーションの設定を行います。（筆者はここが完了していないことに気づかず悪戦苦闘してました。）「プロジェクトの概要」の右の歯車を押下し設定画面を開き、「デフォルトの GCP リソース ロケーション」の設定を完了させましょう。開いたモーダルの下部に表示される「完了」ボタンをそのまま押下すると完了します。

認証機能の実装

続いて認証機能を実装します。誰でもDB操作が行えてしまうとセキュリティ面で問題があるため、認証されたユーザーのみが操作を行えるように設定する必要があります。Firebaseではいくつかの認証方法がありますが、今回はメールとパスワードによる認証を実装していきます。
左のツールバーの「構築」内にある「Authentication」を押下し、画面に表示された「始める」ボタンを押下しましょう。画面中央にひょうじされた「メール / パスワード」ボタンを押下し、「有効にする」トグルにチェックをいれて保存します。これでメールとパスワードによる認証機能が設定されました。

まずはログイン情報を保持するためのContextを作成します。「stores」配下に「auth」ディレクトリを作成し、「AuthProvider.tsx」と「useAuth.tsx」を作成し、providerに追加しましょう。

useAuth.tsx

import { useState, createContext, useContext } from 'react'
import { getAuth, createUserWithEmailAndPassword, signInWithEmailAndPassword, User } from 'firebase/auth'

export type AuthInfo = {
  email: string
  password: string
}

export type UseProvideAuth = {
  // ログイン情報
  user: User | null
  // サインイン処理
  signin: (signInfo: AuthInfo) => void
  // サインアウト処理
  signout: () => void
  // サインアップ処理
  signup: (signInfo: AuthInfo) => void
}

const initialState: UseProvideAuth = {
  user: null,
  signin: () => false,
  signout: () => { },
  signup: () => false
}

/**
 * コンテクストオブジェクト
 */
export const authContext = createContext(initialState)

/**
 * コンテクストオブジェクトの現在値を返すフック
 */
export const useAuth = (): UseProvideAuth => useContext(authContext)

/**
 * コンテキストオブジェクトを更新するフック
 * @returns
 */
export const useProvideAuth = (): UseProvideAuth => {
  const [user, setUser] = useState<User | null>(null)
  const auth = getAuth()

  // サインイン
  const signin = (signInfo: AuthInfo) => {
    const getUserInfo = async (): Promise<void> => {
      const userData = (await signInWithEmailAndPassword(auth, signInfo.email, signInfo.password)).user
      setUser(userData)
    }
    void getUserInfo()
  }

  // サインアウト
  const signout = () => {
    const resetUserInfo = async (): Promise<void> => {
      void await auth.signOut()
      setUser(null)
    }

    void resetUserInfo()
  }

  // サインアップ
  const signup = (signInfo: AuthInfo) => {
    const addUserInfo = async (): Promise<void> => {
      const userData = (await createUserWithEmailAndPassword(auth, signInfo.email, signInfo.password)).user
      setUser(userData)
    }

    void addUserInfo()
  }

  return {
    user,
    signin,
    signout,
    signup
  }
}

DB登録処理の実装

長かったReactstiniもこれで最後の工程です。DB登録処理を実装していきましょう。

初期設定時に少し触れましたが、Firebaseには「Cloud Firestore」というデータベースが存在し一定のデータ量であれば無料で使用することができます。このデータベースは「ドキュメント指向データベース」のため、MySQLやOracleのような「リレーショナルデータベース」とは異なります。詳しくはこちらを参考に理解を深めて頂ければと思いますが、Javascriptで扱うオブジェクトと同じような構造を取ると考えると理解しやすいかと思います。

今回の修正は「stores/eventResources/useEventResources.tsx」のみです。 

import { createContext, useContext, useEffect, useState } from 'react'
import { doc, DocumentData, getDoc, QueryDocumentSnapshot, setDoc, SnapshotOptions, Timestamp } from 'firebase/firestore'
import { useAuth } from '../auth/useAuth'
import { db } from '@/firebase'

type EventResouce = {
  id?: string
  title: string
  start: Date
  end: Date
}
type UseProviderEventResources = {
  eventResouces: EventResouce[]
  addEventResouce: (values: EventResouce) => void
  editEventResouce: (values: EventResouce) => void
  deleteEventResource: (id: string) => void
}

const initialState: UseProviderEventResources = {
  eventResouces: [],
  addEventResouce: () => { },
  editEventResouce: () => { },
  deleteEventResource: () => { }
}

/**
 * コンテクストオブジェクト
 */
export const eventContext = createContext(initialState)

/**
 * コンテクストオブジェクトの現在値を返すフック
 */
export const useEventResources = (): UseProviderEventResources => useContext(eventContext)

/**
 * 型整合用Converter定義
 */
type EventResouceDoc = Omit<EventResouce, 'start | end'> & {
  start: Timestamp
  end: Timestamp
}
type EventDataWrapper<T> = {
  eventResources: T[]
}
type EventConverter = {
  toFirestore: (data: EventDataWrapper<EventResouce>) => DocumentData
  fromFirestore: (snapshot: QueryDocumentSnapshot<EventDataWrapper<EventResouceDoc>>, options: SnapshotOptions) => EventDataWrapper<EventResouce>
}
const eventConverter = (): EventConverter => ({
  toFirestore: (data) => data,
  fromFirestore: (snapshot, options) => {
    const eventDatas: EventResouce[] = snapshot.data(options).eventResources.map((item) => ({
      ...item,
      start: item.start.toDate(),
      end: item.end.toDate()
    }))

    return {
      eventResources: eventDatas
    }
  }
})

export const useProviderEventResources = (): UseProviderEventResources => {
  const [eventResources, setEventResources] = useState<EventResouce[]>([])
  const [isGetEventResouces, setIsGetEventResources] = useState(false)
  const { user } = useAuth()

  const rootDocRef = user ? doc(db, 'eventResources', user.uid).withConverter(eventConverter()) : null

  /**
   * イベントデータ取得処理
   */
  useEffect(() => {
    const getEventResources = async () => {
      if (rootDocRef) {
        const eventResourcesDocData = (await getDoc(rootDocRef)).data()
        eventResourcesDocData?.eventResources && setEventResources(eventResourcesDocData.eventResources)
        setIsGetEventResources(true)
      }
    }

    if (!isGetEventResouces) {
      void getEventResources()
    }
  }, [user, setEventResources, isGetEventResouces, setIsGetEventResources, rootDocRef])

  /**
   * 登録処理
   * @param event
   */
  const addEventResouce = async (event: EventResouce) => {
    if (rootDocRef) {
      const id = Math.random().toString(32).substring(2)

      await setDoc(rootDocRef, {
        eventResources: [
          ...eventResources, {
            ...event,
            id
          }
        ]
      })

      // データ再取得
      setIsGetEventResources(false)
    }
  }

  /**
   * 更新処理
   * @param event
   */
  const editEventResouce = async (event: EventResouce) => {
    if (rootDocRef) {
      const data = eventResources.map((item) => item.id === event.id ? event : item)

      await setDoc(rootDocRef, {
        eventResources: data
      })

      // データ再取得
      setIsGetEventResources(false)
    }
  }

  /**
   * 削除処理
   * @param id
   */
  const deleteEventResource = async (id: string) => {
    if (rootDocRef) {
      const data = eventResources.filter(item => item.id !== id)

      await setDoc(rootDocRef, {
        eventResources: data
      })

      // データ再取得
      setIsGetEventResources(false)
    }
  }

  return {
    eventResouces: eventResources,
    addEventResouce,
    editEventResouce,
    deleteEventResource
  }
}

まず追加された点では「型整合用Converter」です。Firestoreから取得するデータと、画面上で登録を行ったデータ間では型に齟齬が発生するケースがあります。今回ではDate型(画面)とTimestamp型(Firestore)です。登録時にはDate型をそのままリクエストするだけで問題のですが、Firestore側で自動的にTimestamp型に変換されてしまい、データを受け取ったデータそのまま使用するとFullCalendarで表示されないという不具合が発生していまします。
そこでConverterにあらかじめ変換処理を記載することで、データ取得時に画面用の型に変換が可能で、齟齬を意識せずにデータを使用できるようになります。

他の機能は基本的に認証機能と同じ流れになってますが、useEffectを使用することでデータの登録・更新・削除実行後に再度データを取得して同期が取れるようにしている点もポイントです。

では、動作確認です。問題なく登録・更新・削除が確認出来たら、再度ログイン画面からログインしなおしてみましょう。先ほど操作したデータがそのまま表示されるようになったかと思います。


では最後にログインユーザーのみがデータの操作を行えるようにルールを設定します。Firebaseコンソールから「ルール」を選択しましょう。

スクリプトに以下のようなルールを設定してください。 

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow create, read, update: if request.auth != null;
    }
    match /eventResources/{userId} {
      allow create, read, update: if request.auth != null;
    }
  }
}

設定が完了したらテストして見ましょう。スクリプト欄の左下辺りの「ルール プレイグラウンド」を開き以下のように設定し、「実行」を押下してください。

シミュレーションタイプ：get
場所：eventResources/{Authenticationから確認できる認証済みユーザーのUID}
認証済み：チェック

画像のように読み取りが許可されていれば設定完了です。

以上でFirestoreの設定は完了です。
